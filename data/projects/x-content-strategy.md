## Executive Summary

Posting strategy for the @cortex_agent X account launch. Four-phase rollout designed to build familiarity and promote Cortex to vibe-coders and developers using AI coding agents.

## Strategy

### Phase 1: Launch (Day 1)
- **Pinned Post** with value prop + install prompt
- CTA: "Grab cortex.md from github.com/dvn66/cortex and follow its instructions."

### Phase 2: Pain Points (Days 2-7, one per day)
- Each post names a specific frustration developers have with AI agents
- Topics: session amnesia, re-proposing rejected ideas, token waste, platform lock-in, inconsistency, compound learning effect
- Install prompt CTA on 2-3 of 6 posts

### Phase 3: Feature Deep-Dives (Days 8-14, every other day)
- Genesis setup, monitoring mode, triggers workflow, multi-session projects
- Platform support list (existing post, updated)
- "System prompts don't learn" comparison (existing post, updated)

### Phase 4: Ongoing
- Engagement posts ("what should we build next?")
- Philosophy posts ("it's just markdown")
- Release announcements (v1.x posts already drafted)
- Reply-to opportunities on AI coding conversations

## Key Decisions

- Install CTA is a prompt, not a GitHub link: "Grab cortex.md from github.com/dvn66/cortex and follow its instructions."
- Target audience: vibe-coders (people building with AI agents via natural language, not CLI-first developers)
- Language: "one file" not "two files" (single-file DNA model)
- All posts under 280 characters (X limit enforced by CMS)
